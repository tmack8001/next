A) HTML Files & Structure - top

A. 1) Basics

All the pages have been divided into four "main" sections: #header, #content, #footer and #footer-bottom.
.container determines the page width and also centralize it. You can adjust this width from css/style.css file.

Simplified example of fullwidth page structure:

<header id="header" class="container clearfix">
    
    [HEADER_CONTENT_HERE]

</header><!-- end #header -->

<section id="content" class="container clearfix">

    [CONTENT_HERE]
        
</section><!-- end #content -->

<footer id="footer" class="clearfix">
    
    <div class="container">
    
        [FOOTER_CONTENT_HERE]

    </div><!-- end .container -->

</footer><!-- end #footer -->

<footer id="footer-bottom" class="clearfix">
    
    <div class="container">
    
        [FOOTER_BOTTOM_CONTENT_HERE]  

    </div><!-- end .container -->

</footer><!-- end #footer-bottom -->

You have also option to split content area into two part, which allow you to have a sidebar.

Simplified example of page structure with a sidebar:

<header id="header" class="container clearfix">
    
    [HEADER_CONTENT_HERE]

</header><!-- end #header -->

<section id="content" class="container clearfix">

    <section id="main">
    
        [MAIN_CONTENT_HERE]

    </section><!-- end #main -->

    <aside id="sidebar">
    
        [SIDEBAR_CONTENT_HERE]
    
    </aside><!-- end #sidebar -->
        
</section><!-- end #content -->

<footer id="footer" class="clearfix">
    
    <div class="container">
    
        [FOOTER_CONTENT_HERE]

    </div><!-- end .container -->

</footer><!-- end #footer -->

<footer id="footer-bottom" class="clearfix">
    
    <div class="container">
    
        [FOOTER_BOTTOM_CONTENT_HERE]  

    </div><!-- end .container -->

</footer><!-- end #footer-bottom -->

A. 1.1) Basics » Header

Logo

If the logo height is more than 86px, you may need to change #header and #logo height settings from the css/style.css file.

Navigation

Navigation is just basic unordered list so nothing special here, except you can use .data-description attribute to describe the page in a few words. And .current obviously indicates the active page.

A. 1.2) Basics » Content

Page Header

You can place your subpage title in .page-header to give it a little style. Available heading classes:

  • .slogan
  • .page-title
  • .page-description
  • .page-subdescription
  • .project-title

And an example:

<header class="page-header">

    <h1 class="page-title"> [TITLE] </h1>

    <hr />

    <h2 class="page-description"> [DESCRIPTION] </h2>
    
</header><!-- end .page-header -->

Columns

You can easily divide your content between columns by using these classes:

  • .one-half
  • .one-third
  • .two-third
  • .one-fourth
  • .three-fourth

You will need to remember to use class .last in the last column of every row, as well as to add div with class .clear to clear all floats. Like for example:

<div class="one-third">

    [CONTENT]
        
</div><!-- end .one-third -->

<div class="one-third">

    [CONTENT]
        
</div><!-- end .one-third -->

<div class="one-third last">

    [CONTENT]
        
</div><!-- end .one-third last -->

<div class="clear"></div>

You can check all sample columns from columns.html file.

A. 2) Homepage Slider

All slides are placed inside of .ss-slider. Remember to keep .slide-bg-image in image tags to avoid any visual issues.

Also, you don't need to have any content in .slide-content div, but you shouldn't remove it, or otherwise pagination arrows may won't work properly.

Here is the basic slide template:

<article class="slide">

    <img src=" [IMAGE_LOCATION] " alt="" class="slide-bg-image" />
    
    <div class="slide-button"> [BUTTON_CONTENT] </div>

    <div class="slide-content"> [SLIDE_CONTENT] </div>
    
</article><!-- end .slide -->

You can also use images in .slide-button. Then you just need to give class .image to .slide-button, so the image will be positioned correctly.

<div class="slide-button image">
	<img src=" [IMAGE] " alt="" />
</div>

You have also option to link slides to another page:

<article class="slide">

    <a href=" [PAGE_LINK] ">

        <img src=" [IMAGE_LOCATION] " alt="" class="slide-bg-image" />
        
        <div class="slide-button"> [BUTTON_CONTENT] </div>
    
    <a/>

    <div class="slide-content"> [SLIDE_CONTENT] </div>
    
</article><!-- end .slide -->

And Fancybox will also work just fine (excluding gallery):

<article class="slide">

    <a href=" [IMAGE_LINK] " class="single-image">

        <img src=" [IMAGE_LOCATION] " alt="" class="slide-bg-image" />
        
        <div class="slide-button"> [BUTTON_CONTENT] </div>
    
    <a/>

    <div class="slide-content"> [SLIDE_CONTENT] </div>
    
</article><!-- end .slide -->

You can find more info about the slider options in the Javascript section.

A. 3.2) Subpages » Portfolio

Portfolio Filter

All projects are placed in #portfolio-items and every project is a single <article>. Every project has data-categories attribute which will be used in filtering them. A single project can has multiple categories and they need to be separated with space. There isn't any naming rules but you should use only lowercase and single word (use dash to replace spaces) names to avoid any possible issues. And you don't need to use class .last with portfolio items, like usually with columns.

Basic template:

<article class="one-fourth" data-categories=" [CATEGORIES] ">

    <img src=" [IMAGE] " alt="">

    <a href="#" class="project-meta">
        <h5 class="title"> [TITLE] </h5>
        <span class="categories"> [CATEGORIES] </span>
    </a>
	
</article><!-- end .one-fourth -->

Example of filter:

<ul id="portfolio-items-filter">

    <li><a data-categories="*">All</a></li>
    <li><a data-categories="animation">Animation</a></li>
    <li><a data-categories="design">Design</a></li>	
    [...]

</ul><!-- end #portfolio-items-filter -->

Star selector (*) is used to select all items (just like in CSS).

Single Project

Single project has its own slider which can contain any content (images, videos etc.). In demo it has images but there isn't any restrictions what you can or what you can't use in it.

Basic slider:

<div class="image-gallery-slider">

    <ul>

        <li>
            <img src=" [IMAGE] " alt="">
        </li>

        [...]

    </ul>
	
</div><!-- end .image-gallery-slider -->

And you can use this very same slider in blog posts as well.

A. 3.3) Subpages » Contact Form

Setting up the contact form is really simple task. All you need to do is to open php/config.php file and replace $your_email_address value with your own one and that's it!

A. 4) PHP Widgets

Just like with the contact form, all the widget settings can be be changed from the php/config.php file. All settings should be pretty self-explanatory.

Both Latest Tweets and Latest Flickr Images will be cached for one hour at the time to prevent excessive downloading (however cache time can be also changed). In case cache files for same reason won't appear in php/cache folder, try to chmod cache folder to 777. If you don't know how to do that, here's an excellent guide: FTP and CHMOD Tutorial.

A. 6) Pricing Tables

Both the pricing tables can have up to 5 columns, and the column count can be easily changed with .col* class. You can also highlight specific columns with .free and .featured classes. But you'll also need to remember to add .featured class to the pricing table itself, or otherwise the table may look a bit weird.

And the basic table structure is basically identical in both ones: Pricing Table > Colum > Header + Features + Footer

Simple pricing table

<section class="simple-pricing-table col4 clearfix">

    <div class="column">

        <div class="header">
            <h2 class="title"> [TITLE] </h2>
            <h3 class="price"><span> [PRICE] </span> [CONTENT] </h3>
            <h5 class="description"> [DESCRIPTION] </h5>
        </div><!-- end .header -->

        <ul class="features">
            <li> [CONTENT] </li>    
            [...]
        </ul><!-- end .features -->

        <div class="footer">
            <a class="button" href="#"> [CONTENT] </a>
        </div><!-- end .footer -->
        
    </div><!-- end .column -->
    
    [...]

</section><!-- end .simple-pricing-table -->

Extended pricing table

Extended pricing table has a small difference when compared to simple version and that's the .features-list. But you don't need to fear it, since it has exactly the same structure as normal column, except .header and .footer are empty.

This next step requires a little extra work (just tiny copy&paste task), but it's totally worth it, because it allows mobile/tablet users to see all the important content as in desktop browsers. So now when your pricing table otherwise done, the last thing is that you'll need to add data-feature attribute to every "normal" column list item and then fill that with the current row's "title", from .features-list.

If you don't have any idea what I'm talking about, that's totally fine! Just check pricing-tables.html file to see full examples of the both pricing tables!

As you can see, theme doesn't have separate CSS file for IE, but unfortunately this doesn't mean that there wouldn't be any IE specific css rules. Theme has three custom class for this purpose: .ie7, .ie8 and .not-ie. .ie7 is just for IE7, .ie8 for IE8 and older and .not-ie is for any other browser except IE8 and older.

You may also wonder what are all those conditional comments in document head:

<!--[if IE 7]>                  <html class="ie7 no-js" lang="en">     <![endif]-->
<!--[if lte IE 8]>              <html class="ie8 no-js" lang="en">     <![endif]-->
<!--[if (gte IE 9)|!(IE)]><!--> <html class="not-ie no-js" lang="en">  <!--<![endif]-->
<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">

In plain English the part above means:

  1. Line 3: Apply this only if browser is IE7
  2. Line 4: Apply this only if browser is IE8 or older
  3. Line 5: Apply this only if browser is anything else but IE8 or older
  4. Line 16-24: Apply these only if browser is not IE6 or older
  5. Line 26-28: Apply these only if browser is IE6 or older
  6. Line 30-32: Apply this only if browser is IE8 or older

C. 5) HTML5 video & audio

HTML5 video

Using HTML5 videos in your site isn't that hard, as you can see in the example below. But have also option to use HMTL5 Video Tag Builder, which generates all the needed stuff automatically. Unfortunately it's not totally perfect for this template, since it's using fixed dimensions.

To make responsive videos, you'll need to remove both width and height attributes and set aspect ratio (data-aspect-ratio) instead. By default it's set to 1.78 (16:9), so if that's fine for you, you can basically just remove that attribute. But in case you need to change it, here's one excellent tool for that task: Aspect Ratio Calculator.

<video class='video-js vjs-default-skin' poster=" [IMG_POSTER_URL] " data-aspect-ratio='1.78' data-setup="{}" controls>
    <source src=" [MP4_VIDEO_URL] " type="video/mp4" />
    <source src=" [WEBM_VIDEO_URL] " type="video/webm" />
    <source src=" [OGV_VIDEO_URL] " type="video/ogg" />
</video>

Remember also to update the location of the self-hosted video-js.swf file in every .html file at line 37.

_V_.options.flash.swf = 'http://demo.samuli.me/smartstart/js/video-js.swf';

Plugin documentation: HTML5 Video Player | Video.js
Some info about the element itself: The video element | HTML5 Doctor

HTML5 audio

Using HTML5 audio isn't that much different, just remember to use .AudioPlayerV1 to enable custom design.

<audio class="AudioPlayerV1" controls>
    <source src=" [MP3_AUDIO_URL] " type="audio/mp3" />
    <source src=" [OGG_AUDIO_URL] " type="audio/ogg" />
</audio>
